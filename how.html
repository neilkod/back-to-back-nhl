How this works

The whole planner is four small, plain functions. Here's what each one does, in the order it runs.

Every team here plays roughly 40 home games a season. If you plot only the home games on a calendar, some nights line up back-to-back — but "on the calendar next to each other" only matters if you could actually get from arena A to arena B in time. That's the one rule driving everything below: a trip is a chain of consecutive-night home games where every hop between arenas is a travel time you're willing to make.

1. The link test

Two games — any two, any nights — are "linkable" at a given tolerance if the drive/train/flight time between their arenas is at or under that tolerance. That's it; travel time is a fixed lookup table between arena pairs, not a distance calculation. 

function isLinkable(a, b, tolerance) {
  const h = getTravelHours(a, b);
  return h !== null && h <= tolerance;
}

A missing table entry means "not reachable at any tolerance" — that's how, for example, the California teams (Kings, Ducks) stay structurally cut off from every Eastern team on this board, at every slider stop.

2. The streak sweep — and the trap in the obvious version

The naive way to find a run: walk the calendar day by day, and keep the streak alive as long as some team playing tonight links to some team playing tomorrow night. That sounds right, but it isn't — because on a night with several games, the specific team your itinerary is riding might not be the team that supplies tomorrow's link.

Concretely: from Jan 8–14 this season, the only linkable game each night is an LA ↔ Anaheim shuttle (0.75h apart). Meanwhile Washington and Boston happen to play games on some of those same calendar dates that would pairwise-link to Jan 15's Islanders game. A naive pairwise test sees that coincidence and calls the whole Jan 8–28 span one 21-night run. But nobody is actually playing Washington or Boston those nights — your itinerary is stuck on the LA/Anaheim shuttle, and Anaheim has no travel-table entry to the Islanders at all. There's no way to actually ride that "run."

The fix is to track a frontier: the actual set of games your itinerary could be sitting at tonight, given how it got here. Tomorrow only extends the streak if at least one of tomorrow's games links to at least one game in tonight's frontier — not to any game that happens to be on the calendar. 

function findRunDateGroups(dateMap, dates, tolerance) {
  const groups = [];
  let currentDates = null, frontier = null;
  for (const date of dates) {
    const dayGames = dateMap[date];
    if (currentDates) {
      const prevDate = currentDates[currentDates.length - 1];
      if (isNextCalendarDay(prevDate, date)) {
        const nextFrontier = dayGames.filter((g) =>
          frontier.some((h) => isLinkable(h.abbr, g.abbr, tolerance)));
        if (nextFrontier.length > 0) {
          currentDates.push(date);
          frontier = nextFrontier;
          continue;
        }
      }
      groups.push(currentDates);
    }
    currentDates = [date];
    frontier = dayGames;
  }
  if (currentDates) groups.push(currentDates);
  return groups.filter((g) => g.length >= 2);
}

With the frontier fix, that span correctly splits into a 7-night West Coast run (Jan 8–14) and a genuine 14-night corridor run (Jan 15–28) — which is the real longest trip on the board, even at the widest "Add Boston" setting.

3. Turning "a run exists" into an actual itinerary

Knowing a run's date range isn't the same as knowing which game to attend each night — some nights have several home games in range, and a choice early in the run can dead-end later. So the planner makes two passes over each run:

Backward pass — starting from the last night and working back, keep only the games that can still reach the end of the run: 

function computeViableGames(runDates, dateMap, tolerance) {
  const n = runDates.length;
  const viable = new Array(n);
  viable[n - 1] = dateMap[runDates[n - 1]].slice();
  for (let i = n - 2; i >= 0; i--) {
    const nextViable = viable[i + 1];
    viable[i] = dateMap[runDates[i]].filter((g) =>
      nextViable.some((h) => isLinkable(g.abbr, h.abbr, tolerance)));
  }
  return viable;
}

Forward pass — walk night to night picking the shortest hop from wherever the itinerary currently is, only choosing among the games the backward pass proved viable. This is what builds the trip card you actually see, and it's also what re-runs (instantly — the dataset is tiny) whenever you swap a night to an alternate team.

4. Doubleheaders

Same night, two teams, both playing at home: flagged as a doubleheader if their start times are at least 4.5 hours apart and their arenas are linkable at the current tolerance — enough gap to catch a matinee, travel, and still make the evening game. 

function findDoubleheader(dayGames, tolerance) {
  for (const g1 of dayGames) for (const g2 of dayGames) {
    const gap = Math.abs(g1.hour - g2.hour);
    if (gap >= 4.5 && isLinkable(g1.abbr, g2.abbr, tolerance)) {
      return { matinee: /* earlier */ g1, evening: /* later */ g2, gap };
    }
  }
}
