How this works

The whole planner is four small functions. Here's what each one does, in order.

Every team here plays about 40 home games a season. Plot just the home games on a calendar and some nights line up back to back. But "next to each other on the calendar" only matters if you can actually get from arena A to arena B in time. That's the whole rule: a trip is a chain of consecutive-night home games, and every hop between arenas has to fit your travel tolerance.

1. The link test

Two games are "linkable" at a given tolerance if the travel time between their arenas is at or under that tolerance. Doesn't matter which two nights. Travel time comes from a fixed lookup table between arena pairs, not a distance calculation. 

function isLinkable(a, b, tolerance) {
  const h = getTravelHours(a, b);
  return h !== null && h <= tolerance;
}

A missing table entry means "not reachable, period." That's why the California teams (Kings, Ducks) never link up with an Eastern team, at any slider setting.

2. The streak sweep (and its trap)

The obvious approach: walk the calendar day by day. Keep the streak alive as long as some team playing tonight links to some team playing tomorrow. Sounds right. It isn't. On a night with several games, the team your itinerary is actually riding might not be the one supplying tomorrow's link.

Here's a real example. From Jan 8 to 14 this season, the only linkable game each night is an LA/Anaheim shuttle (45 minutes apart). Washington and Boston happen to play on some of those same dates, and those games would pairwise-link to the Islanders game on Jan 15. A naive test sees that coincidence and calls the whole Jan 8-28 span one 21-night run. But nobody's actually playing Washington or Boston those nights. Your itinerary is stuck on the LA/Anaheim shuttle, and Anaheim has no travel entry to the Islanders at all. That "run" doesn't exist.

The fix: track a frontier. That's the actual set of games your itinerary could be at tonight, given how it got there. Tomorrow only extends the streak if one of tomorrow's games links to something in tonight's frontier, not to any game that happens to be on the calendar that day. 

function findRunDateGroups(dateMap, dates, tolerance) {
  const groups = [];
  let currentDates = null, frontier = null;
  for (const date of dates) {
    const dayGames = dateMap[date];
    if (currentDates) {
      const prevDate = currentDates[currentDates.length - 1];
      if (isNextCalendarDay(prevDate, date)) {
        const nextFrontier = dayGames.filter((g) =>
          frontier.some((h) => isLinkable(h.abbr, g.abbr, tolerance)));
        if (nextFrontier.length > 0) {
          currentDates.push(date);
          frontier = nextFrontier;
          continue;
        }
      }
      groups.push(currentDates);
    }
    currentDates = [date];
    frontier = dayGames;
  }
  if (currentDates) groups.push(currentDates);
  return groups.filter((g) => g.length >= 2);
}

With the frontier fix, that span splits correctly: a 7-night West Coast run (Jan 8-14) and a genuine 14-night corridor run (Jan 15-28). Widen the tolerance further, to the "Add Boston" setting, and this same corridor run stretches to 17 nights (Jan 12-28).

3. Turning "a run exists" into an actual itinerary

Knowing a run's date range isn't the same as knowing which game to attend each night. Some nights have several home games in range, and picking wrong early in the run can dead-end you later. So the planner makes two passes over each run.

Backward pass. Start from the last night and work backward, keeping only the games that can still reach the end of the run. 

function computeViableGames(runDates, dateMap, tolerance) {
  const n = runDates.length;
  const viable = new Array(n);
  viable[n - 1] = dateMap[runDates[n - 1]].slice();
  for (let i = n - 2; i >= 0; i--) {
    const nextViable = viable[i + 1];
    viable[i] = dateMap[runDates[i]].filter((g) =>
      nextViable.some((h) => isLinkable(g.abbr, h.abbr, tolerance)));
  }
  return viable;
}

Forward pass. Walk night to night, picking the shortest hop from wherever the itinerary currently sits, choosing only from games the backward pass proved viable. This builds the trip card you actually see. It also re-runs instantly (the dataset is tiny) whenever you swap a night to an alternate team.

4. Doubleheaders

Same night, two teams, both playing at home. It's flagged as a doubleheader if their start times are at least 4.5 hours apart and their arenas are linkable at the current tolerance. That's enough gap to catch a matinee, travel, and still make the evening game. 

function findDoubleheader(dayGames, tolerance) {
  for (const g1 of dayGames) for (const g2 of dayGames) {
    const gap = Math.abs(g1.hour - g2.hour);
    if (gap >= 4.5 && isLinkable(g1.abbr, g2.abbr, tolerance)) {
      return { matinee: /* earlier */ g1, evening: /* later */ g2, gap };
    }
  }
}
